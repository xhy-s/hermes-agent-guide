# 核心概念

## 1.1 Agent 对话循环机制

Hermes 的核心是 **AIAgent** 类（定义在 `run_agent.py`），它实现了标准的 Agent 对话循环：

```
┌─────────────────────────────────────────────────────────────────┐
│                     AIAgent.run_conversation()                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 初始化 (__init__)                                            │
│     - 加载工具 (model_tools._discover_tools)                    │
│     - 初始化 SessionDB (hermes_state.SessionDB)                 │
│     - 设置迭代预算 (max_iterations, iteration_budget)            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 消息循环 (while budget > 0)                                  │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  2.1 构建请求 (build_request)                               ││
│  │      - 组装 system prompt                                   ││
│  │      - 添加上下文引用 (context_references)                  ││
│  │      - 上下文压缩 (context_compressor)                      ││
│  └─────────────────────────────────────────────────────────────┘│
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  2.2 调用模型 (call_llm)                                   ││
│  │      - 发送请求到 API (OpenAI/Anthropic 兼容)               ││
│  │      - 处理流式/非流式响应                                   ││
│  └─────────────────────────────────────────────────────────────┘│
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  2.3 处理响应 (handle_response)                             ││
│  │      - 解析 tool_calls                                      ││
│  │      - 执行工具或返回消息                                    ││
│  └─────────────────────────────────────────────────────────────┘│
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  2.4 工具执行 (handle_function_call)                        ││
│  │      - 从 registry 获取处理器                               ││
│  │      - 执行工具并收集结果                                    ││
│  │      - 将结果追加到消息历史                                  ││
│  └─────────────────────────────────────────────────────────────┘│
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  2.5 预算管理 (budget--)                                    ││
│  │      - 检查迭代次数                                          ││
│  │      - 触发上下文压缩（如需要）                              ││
│  └─────────────────────────────────────────────────────────────┘│
│                              │                                   │
│              ┌───────────────┴───────────────┐                 │
│              ▼                               ▼                 │
│     budget > 0                          budget <= 0              │
│     继续循环                              退出循环               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 返回结果 (return result)                                     │
│     - 生成最终响应                                               │
│     - 保存会话到 SessionDB                                       │
└─────────────────────────────────────────────────────────────────┘
```

## 1.3 Memory 与 Session 管理

### 1.3.1 SessionDB 架构

**文件**: `hermes_state.py`

```python
class SessionDB:
    """
    SQLite + FTS5 实现的会话存储
    支持全文搜索、语义搜索
    """
    def __init__(self, db_path: str):
        self.conn = sqlite3.connect(db_path)
        self._setup_fts5()  # 全文搜索索引

    def add_message(self, session_id: str, message: dict): ...
    def search(self, query: str) -> List[dict]: ...  # FTS5 搜索
    def semantic_search(self, query: str) -> List[dict]: ...  # 语义搜索
```

### 1.3.2 多层记忆架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     Hermes 记忆层次                              │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: 上下文窗口 (Context Window)                            │
│  - 最近 N 条消息                                                │
│  - 由模型上下文长度决定 (如 200k tokens)                        │
│  - 自动管理，无需手动                                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ (超过阈值时触发压缩)
┌─────────────────────────────────────────────────────────────────┐
│  Layer 2: SessionDB (短期记忆)                                   │
│  - SQLite 数据库                                                 │
│  - 当前会话所有消息                                              │
│  - FTS5 全文搜索                                                │
│  - 路径: ~/.hermes/sessions.db                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ (用户明确保存时)
┌─────────────────────────────────────────────────────────────────┐
│  Layer 3: Memory Tool (长期记忆)                                 │
│  - 语义存储                                                      │
│  - 跨会话持久化                                                  │
│  - 可被 Agent 主动查询/写入                                      │
│  - 工具: memory_tool.py                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ (可选集成)
┌─────────────────────────────────────────────────────────────────┐
│  Layer 4: Honcho (用户建模)                                      │
│  - dialectic 用户建模                                            │
│  - 跨会话学习用户偏好                                            │
│  - honcho_integration/                                           │
└─────────────────────────────────────────────────────────────────┘
```

## 1.4 Skills 系统

### 1.4.1 Skill 定义

Skill 是一个**可打包的功能单元**，包含：
- **元数据**: 名称、描述、版本、作者
- **指令**: system prompt 片断
- **工具**: 关联的工具集
- **触发条件**: 何时激活

```
┌─────────────────────────────────────────────────────────────────┐
│                        Skill 结构                                │
└─────────────────────────────────────────────────────────────────┘

optional-skills/
└── example-skill/
    ├── DESCRIPTION.md       # Skill 元数据
    ├── SKILL.md            # 指令定义
    └── references/          # 参考文档
        ├── api.md
        └── examples.md
```

### 1.4.2 Skill 生命周期

```
┌─────────────────────────────────────────────────────────────────┐
│                    Skill 生命周期                                │
└─────────────────────────────────────────────────────────────────┘

1. 定义 (Definition)
   └─ 在 optional-skills/ 或 skills/ 中定义

2. 注册 (Registration)
   └─ skills_tool.py 扫描并注册

3. 激活 (Activation)
   └─ Agent 根据上下文自动激活或用户手动触发
   └─ 激活时将 SKILL.md 内容注入 system prompt

4. 执行 (Execution)
   └─ Agent 使用 Skill 关联的工具

5. 学习 (Learning)
   └─ Agent 从经验中改进 Skill（自改进机制）
```

## 1.5 模型路由与适配

### 1.5.1 多模型支持

Hermes 通过**适配器模式**支持多种模型：

```
┌─────────────────────────────────────────────────────────────────┐
│                    模型适配层                                    │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  agent/anthropic_adapter.py                                      │
│  - Anthropic API (Claude) 专用逻辑                              │
│  - Prompt caching                                                │
│  - 工具使用优化                                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  agent/auxiliary_client.py                                      │
│  - 辅助模型客户端                                                │
│  - Vision 处理 (图片理解)                                        │
│  - 摘要生成                                                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  agent/smart_model_routing.py                                   │
│  - 智能模型路由                                                  │
│  - 根据任务类型选择模型                                          │
└─────────────────────────────────────────────────────────────────┘
```

### 1.5.2 支持的模型提供商

| 提供商 | 适配器 | 支持模型 |
|--------|--------|---------|
| OpenAI | OpenAI API | GPT-4, GPT-3.5 |
| Anthropic | anthropic_adapter | Claude 3.5, Claude 3 |
| OpenRouter | OpenAI 兼容 | 200+ 模型 |
| MiniMax | 自定义 | MiniMax M2.7 |
| Z.ai/GLM | 自定义 | GLM 系列 |
| 自定义端点 | OpenAI 兼容 | 任意兼容 API |
